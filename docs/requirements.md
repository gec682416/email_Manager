## 一、项目背景

随着邮件、会议、即时通讯等办公信息持续增加，用户每天需要处理大量重复性的事务，包括：

* 阅读邮件
* 提取关键信息
* 安排会议
* 创建待办事项
* 回复邮件
* 跟踪项目进展

传统邮箱和日历工具仅提供“信息展示能力”，缺乏对事务的主动理解与执行能力。

本项目旨在构建一个基于 LLM 的 AI 邮件/会议管家 Agent，通过接入邮箱与日历系统，实现：

* 自动理解邮件内容
* 自动识别会议与待办事项s
* 自动生成会议日历
* 自动生成邮件回复草稿
* 提供事务级别的智能辅助能力

最终目标是构建一个“个人工作流操作系统（Personal Workflow OS）”。

# 二、项目目标

## MVP（第一阶段）目标

实现：

```text
收取邮件
→ AI 理解邮件内容
→ 提取会议信息
→ 自动创建会议日历
→ 生成会议摘要/回复建议
```

要求：

* 用户无需手动整理会议
* 支持自然语言时间解析
* 支持会议提醒
* 支持邮件摘要
* 支持低成本运行

# 三、系统整体架构

```text
网易邮箱（IMAP）
        ↓
邮件拉取服务
        ↓
邮件解析模块
        ↓
LLM Agent
 ├── 邮件摘要
 ├── 意图识别

业务执行层

        ↓
```

# 四、核心功能模块（重点）

# 4.1 邮件接收模块（IMAP）

## 功能描述

通过 IMAP 协议定时拉取用户邮箱中的邮件。

支持：

* 未读邮件拉取
* 指定时间范围邮件同步
* 邮件增量同步
* 邮件状态同步
* 邮件附件读取

---

## 技术方案

使用：

```text
IMAP + SSL
```

网易邮箱配置：

```text
imap.163.com:993
```

认证方式：

```text
邮箱账号 + 客户端授权码
```

---

## 功能细节

### 邮件同步

支持：

* 全量首次同步
* 基于 UID 的增量同步

避免重复拉取。

---

### 邮件解析

解析：

* 发件人
* 收件人
* 标题
* 正文
* HTML 内容
* 附件
* 时间

---

### 异常处理

包括：

* IMAP 连接断开
* 登录失效
* 网易邮箱频控
* 邮件格式异常

---

# 4.2 邮件 AI 理解模块（核心模块）

## 功能描述

使用 LLM 对邮件进行语义理解。

这是整个 Agent 的核心能力。

---

## 功能能力

### 1. 邮件摘要

输入：

```text
长邮件 / 邮件线程
```

输出：

```text
摘要
关键决策
待处理事项
```

---

### 2. 邮件分类

识别：

* 会议
* 待办
* 广告
* 通知
* 客户邮件
* 面试邮件
* 财务邮件

---

### 3. 意图识别

判断：

```text
用户是否需要：
- 创建会议
- 回复邮件
- 创建 TODO
- 跟进客户
- 修改日程
```

---

### 4. 时间解析

识别：

```text
下周三下午两点
明天上午
5 月 8 日 14:00
```

转换为标准时间格式。

---

### 5. TODO 提取

例如：

```text
请 Jonathan 周五前提交方案
```

提取：

```text
任务：提交方案
截止时间：周五
负责人：Jonathan
```

---

## 技术实现

推荐：

```text
LLM + Prompt Engineering
```

后期可增加：

```text
RAG
Memory
Tool Calling
```
